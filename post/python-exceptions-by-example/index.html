Python Exceptions by Example

Python try/except(/finally/else) blocks are a very powerful tool. It allows one to run code which could fail and deal with a failure errors (exception) in a targeted way.

This is how most people use them when they first learn about try/except blocks:

System Message: WARNING/2 (<string>, line 7)

Cannot analyze code. Pygments package not found.

.. code-block:: python

        try:
                # do a dumb thing
        [...]
    except:
        # Oh glob it didn't work -- put out the fire
        [...]

While this gets the job done and uses the tool, the pythonic [1] way to do things is to deal with these types of errors in a more fine-grained way:

System Message: WARNING/2 (<string>, line 19)

Cannot analyze code. Pygments package not found.

.. code-block:: python

        try:
        # to do a thing
        [...]
        except ExceptionError as e:
        # oops something happen
        [...]
    exception OtherExceptionError as e:
        # oops a different something happened
        [...]
    else:
        # No exceptions occurred, continue like normal.
        [...]

For example:

System Message: WARNING/2 (<string>, line 36)

Cannot analyze code. Pygments package not found.

.. code-block:: python

        for x in range(5,-1,-1):
                try:
                        y = 1/x  # Performs calculation
                except ZeroDivisonError as e:
                        print(e) # Prints error to console
            # Alternatively you can just
            print('And then the world imploded...')
                else:
                        print(y) # Prints 1/x to console

This is a very exciting way to deal with problems as they arise, you get to do pretty dumb things (like divide by 0, but it requires that you know about the exception types and what they mean.

Finding New Exceptions

Let's say we have the following script:

System Message: WARNING/2 (<string>, line 58)

Cannot analyze code. Pygments package not found.

.. code-block:: python

    for x in range(5,-1,-1):
        y = 1/x
        print(y)

When run it produces the following output:

0.2
0.25
0.3333333333333333
0.5
1.0
Traceback (most recent call last):
  File "exception.py", line 2, in <module>
    y = 1/x
ZeroDivisionError: division by zero

We're sticking with the divide by zero problem since it's pretty easy one to get your head around and an easy mistake to make.

In this example the Ze­ro­Di­vi­sion­Error is the python exception being thrown. This is how all python exceptions are displayed when a script is run, they're almost always the last line, or the last line of an error block.
